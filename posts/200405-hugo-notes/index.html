Quick Start

hugo new site <site-name> # new site
git init; git submodule add <url> themes/theme-name # add a theme from remote
hugo new <CAT>/<FILE>.<FORMAT> # new article, mainly .md
hugo server -D # local server
hugo -D # build in ./public/, 'D' for deploy

Drafts do not get deployed; once you finish a post, update the header of the post to say draft: false.

Notes for submodule

My blog uses the theme Mainroad, and I added some additional features, for example, tap-to-top, word count, archive, etc. If the new features can be added outside the theme, then, we can keep the theme repository clean. If it has to be put in the theme file, like the tap-to-top module, we shall open a new branch in the submodule directory, in order to easily upgrade theme later.

Notes for taxonomies

My content folder has 3 subfolders and 1 file:

  • posts/ for blog pages
  • publications/ for research papers publishing
  • archives/ for Archives page
  • about.md for About Me page

Every article shall have categories and tags for taxonomies. As for categories, we should list almost all categories we will use before hand. For me, I will use

  • Programing
  • Math
  • Distributed Optimization
  • etc.

Moreover, I usually name a file with date first, like yymmdd-filename.md.
